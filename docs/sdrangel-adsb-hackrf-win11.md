# SDRangel + HackRF One 在 Windows 11 上接收 ADS‑B（1090MHz）速记

适用场景：HackRF One 已能被 Windows 11 识别（设备管理器能看到 `HackRF One (VID_1D50&PID_6089)`），并且 SDRangel 已安装。

## 0. 先确认硬件“真连上了”

- HackRF One 插上 USB 后，建议至少看到 `3V3 / 1V8 / RF / USB` 四个灯亮。
- 如果设备管理器出现“未知 USB 设备(端口重置失败) / Code 43”，优先排查线缆（很多 micro‑USB 线只有供电没数据，或信号质量差）。

## 1. 在 SDRangel 里切到 HackRF 输入

目标：左侧设备面板标题显示为 `HackRF[0]`（或类似），不是 `FileInput`。

## 2. 设置 HackRF 基本参数（左侧 HackRF 面板）

推荐起步值（先让它“出数据”，再慢慢收敛）：

- 中心频率：`1090.000 MHz`（即 `1,090,000 kHz`）
- 采样率（SR）：`2.4 MSPS`（≥ 2 MSPS 基本都可）
- `DC`：开
- `IQ`：开
- `BBF`：开（保持默认也行）
- `RF Amp`：开（先开，后面如果过载再关）
- `LNA`：`32 dB`
- `VGA`：`24 dB`

### 参数/开关解释（看这一段就够用）

- **中心频率（Center Frequency）**：把 HackRF 的“调谐频点”对准 `1090MHz`（ADS‑B 主频）。你看到的 `1,090,000 kHz` 就是 `1090.000 MHz`。
- **采样率（SR / Sample Rate）**：每秒采集多少个 IQ 样本。ADS‑B 的信道带宽需要足够的采样率来覆盖；经验上 **≥ 2 MS/s** 更稳，你用 `2.4 MSPS` 很常见。
- **`DC`（DC removal / DC blocker）**：直流偏置抑制。实际硬件/混频器会带来直流分量，在频谱中心形成一根“DC 尖峰”；打开它能减弱尖峰，避免把解调器的动态范围吃掉。
- **`IQ`（IQ imbalance correction）**：I/Q 失衡校正。I/Q 幅度/相位不一致会产生“镜像”与杂散；打开它能让频谱更干净，提升解调稳定性。
- **`BBF`（Baseband Filter）**：基带滤波器/带宽限制。它决定从调谐中心频率附近“放行”多宽的基带频谱（例如你界面上的 `BBF 1.75M`）。一般保持默认即可：太窄会把 ADS‑B 脉冲能量削掉；太宽则引入更多噪声。
- **`RF Amp`（RF front-end amplifier）**：射频前端放大器开关。打开后能提升灵敏度，但在强信号/城市干扰较多时可能过载（表现为噪声地板整体抬高、到处都是假峰、解调变差）。默认“先开出数据”，再根据频谱与误报情况决定是否关闭。
- **`LNA`（Low Noise Amplifier gain）**：低噪声放大器增益，位置更靠近射频前端，通常对弱信号更有帮助；太大也会推高过载风险。
- **`VGA`（Variable Gain Amplifier gain）**：可变增益放大器增益（多在后级/基带或 IF 相关链路），用于把信号幅度“抬到合适范围”；同样过大可能导致饱和/失真。

实用口诀（防止越调越糊）：
- 频谱/瀑布里能看到明显 ADS‑B 脉冲，但列表没数据：优先调 `Threshold`，再微调 `LNA/VGA`。
- 噪声地板突然抬高、满屏杂散、误报很多：优先关 `RF Amp` 或下调 `LNA/VGA`。

说明：
- `DC`/`IQ` 打开是为了抑制直流尖峰与 IQ 不平衡，避免把解调器“糊住”。
- 增益不是越大越好；但在室内/普通天线条件下，先按上面的值通常更容易看到包。

## 3. 启动接收（RX）

- 点左侧 HackRF 面板的绿色开始按钮（变成“Stop”表示正在接收）。
- 在频谱/瀑布图里观察 1090MHz 附近是否有短促的竖线/点状脉冲（这是 ADS‑B 包的典型表现）。

## 4. 添加 ADS‑B 解调通道

SDRangel 的“Add channel(s)”不在菜单里，通常在设备窗口顶部那排小图标中：

- 点 `Add channels`（或 `+` / “Add channel(s)”图标）
- 在弹窗里选择 `ADS-B Demodulator`
- `Apply` → `Close`

如果看不到新增的解调器窗口：
- 用 `Show all channels`（显示所有通道）按钮把窗口找出来
- 或用 `Stack windows`/自动排版把窗口堆叠出来

## 5. ADS‑B 解调器关键参数

在 `ADS-B Demodulator` 面板里：

- `Δf`：保持 `0 Hz`（除非你刻意把中心频率偏到 1090.5MHz 来躲 DC）
- `BW`：保持默认（常见是约 `2.6M`）
- `Threshold`：如果“没任何数据”，把阈值先降到 **3~4** 试；如果误报很多再升回去

经验：
- “能看到瀑布图有脉冲但列表没飞机” → 多半是阈值偏高或增益偏低。
- “瀑布图完全没脉冲” → 多半是天线/位置（靠窗、室外、1090 专用天线会提升巨大）。

## 6. 地图（可选）

添加地图 Feature（用于显示飞机位置）：
- `Create a new feature` → `Map`
- 在 `Preferences` 里设置 `My Position`（你的位置），地图会以此作为 home station

## 7. 一键排障（从现象倒推）

- 设备管理器都看不到 HackRF / 只出现“端口重置失败”：
  - 换线（优先“确认能传数据”的线）、换 USB 口、避免 Hub/扩展坞
- SDRangel 里还是 `FileInput`：
  - 没切换到 HackRF 输入设备
- 频谱只有正中间一根很高的尖峰、其余看不见包：
  - 先开 `DC` + `IQ`；必要时把中心频率偏移（1090.5MHz + `Δf=-500kHz`）
- 列表没数据：
  - 降 `Threshold`，适当提高 `LNA/VGA`，确认 `RF Amp` 状态
